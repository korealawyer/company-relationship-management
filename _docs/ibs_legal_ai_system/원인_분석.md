# 프롬프트 절반 축소 후에도 1800자 미달 원인 분석

## 📊 현재 상황
- **프롬프트 길이**: 2181자 (이전 대비 절반 이하로 축소 ✅)
- **초안 길이**: 1269자 (공백 제외 970자) ❌
- **요약된 컨텍스트**: 1510자
- **원본 컨텍스트**: 5449자

## 🔍 핵심 문제점

### 1. **프롬프트 구성 비율 문제** (가장 심각)
```
프롬프트(2181자) 구성:
├─ 길이 요구사항: ~100자 (4.6%)
├─ 기본 지시사항: ~200자 (9.2%)
├─ 인용 필수 법령/판례: ~100자 (4.6%)
├─ 주제: ~50자 (2.3%)
└─ 요약된 컨텍스트: 1510자 (69.2%) ⚠️ 문제!
```

**문제**: 요약된 컨텍스트가 프롬프트의 **69%**를 차지합니다. LLM이 요약본을 읽고 그에 기반해 짧게 작성하는 경향이 있습니다.

### 2. **요약본의 특성**
- 요약본은 이미 핵심만 추출한 간결한 형태
- LLM이 요약본을 읽으면 "이미 요약되어 있으니 짧게 작성해야 한다"고 인식할 수 있음
- 요약본의 구조가 이미 정리되어 있어 확장할 필요성을 느끼지 못함

### 3. **길이 요구사항 강조 부족** (로그 기준)
로그를 보면 실제 프롬프트는:
```
"**최우선**: 최소 1800자 이상 (목표: 1800자). 미달 시 거부."
```
이 형식은 수정한 코드의 "🚨 **절대 필수 - 최우선 요구사항** 🚨" 형식이 아닙니다.

**가능한 원인**:
- 코드 변경이 반영되지 않았거나
- 캐시된 이전 버전이 실행되었거나
- 다른 경로로 프롬프트가 생성되었을 수 있음

### 4. **프롬프트 구조 문제**
현재 프롬프트 구조:
```
1. 길이 요구사항 (맨 위)
2. 기본 지시사항
3. 인용 필수 법령/판례
4. 주제
5. 참고 문서 (요약된 컨텍스트 1510자) ← 여기서 끝나버림
6. 마지막 길이 요구사항 (하지만 이미 LLM이 생성 시작)
```

**문제**: 요약된 컨텍스트가 프롬프트의 끝부분에 위치하여, LLM이 이를 읽고 생성할 때 이미 컨텍스트의 길이에 맞춰 짧게 작성하기 시작합니다.

### 5. **max_tokens는 충분함**
- `max_tokens = 4096`으로 설정되어 있어 토큰 제한은 문제 없음
- 문제는 LLM이 **짧게 생성하려는 의도**가 아니라, **요약본에 기반해 자연스럽게 짧게 작성**하는 것

## 💡 해결 방안

### 방안 1: 요약된 컨텍스트를 더 짧게 (즉시 적용 가능)
- 현재: 최소 1200자 (원본의 20%)
- 제안: 최소 800자 (원본의 15%)로 축소
- 효과: 프롬프트에서 요약본 비율을 69% → 45%로 감소

### 방안 2: 프롬프트 구조 재배치 (권장)
- 길이 요구사항을 **요약된 컨텍스트 바로 앞**에 배치
- 요약된 컨텍스트 다음에 **다시 한 번 길이 요구사항 강조**
- 구조:
  ```
  1. 기본 지시사항
  2. 주제
  3. 🚨 길이 요구사항 강조 🚨
  4. 참고 문서 (요약된 컨텍스트)
  5. 🚨 길이 요구사항 재강조 + 확장 지시 🚨
  ```

### 방안 3: 요약 프롬프트 개선
- 요약 프롬프트에 "요약본을 읽고 **1800자 이상의 상세한 콘텐츠를 작성할 수 있도록** 충분한 정보 제공" 명시
- 요약본 자체에 "이 요약본을 바탕으로 **상세하게 확장**하여 작성" 지시 포함

### 방안 4: 프롬프트에 확장 지시 추가
- 요약된 컨텍스트 다음에:
  ```
  **중요**: 위 요약본은 참고용입니다. 각 내용을 **최소 3배 이상 확장**하여 
  상세하게 설명하세요. 예를 들어:
  - 법령 조문: 구성요건을 각각 상세히 설명
  - 판례: 사실관계를 단계별로 설명, 법원 판단을 논리적으로 설명
  - 개념: 예시와 함께 구체적으로 설명
  ```

### 방안 5: 시스템 프롬프트 강화
- 시스템 프롬프트에 길이 요구사항을 더 강조
- "항상 최소 1800자 이상 작성"을 핵심 원칙으로 추가

## 🎯 권장 조합
1. **즉시**: 요약된 컨텍스트를 800자로 축소
2. **즉시**: 프롬프트 구조 재배치 (길이 요구사항을 요약본 앞뒤에 배치)
3. **즉시**: 요약본 다음에 확장 지시 추가
4. **검토**: 시스템 프롬프트 강화

이 조합으로 프롬프트에서 요약본 비율을 45% 이하로 낮추고, 길이 요구사항을 더 강조할 수 있습니다.
