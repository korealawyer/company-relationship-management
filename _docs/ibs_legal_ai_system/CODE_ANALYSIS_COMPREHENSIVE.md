## 프로젝트 개요

### 프로젝트 목적

IBS 법률 AI 시스템은 **RAG(Retrieval-Augmented Generation)** 기술을 활용한 법률 정보 질의응답 시스템입니다. 법령, 판례, 절차 매뉴얼 등 다양한 법률 문서를 벡터 데이터베이스에 저장하고, 의미 기반 검색과 LLM을 결합하여 정확한 법률 정보를 제공합니다.

### 핵심 특징

- **10가지 법률 문서 타입 지원**: 법령, 판례, 절차 매뉴얼, 실무 매뉴얼, 사건 유형, 템플릿, 양형기준, FAQ, 키워드 맵핑, 스타일 문제
- **하이브리드 검색**: 벡터 검색과 키워드 검색을 결합한 고성능 검색
- **LangGraph 기반 워크플로우**: 구조화된 RAG 파이프라인
- **콘텐츠 생성**: 블로그, 기사, 의견서 등 자동 생성
- **세션 관리**: 멀티 턴 대화 지원 (Redis 선택사항)
- **스트리밍 응답**: Server-Sent Events 기반 실시간 응답
- **필터링 관리**: 통합된 FilterManager로 필터링 로직 중앙화 ⭐
- **임베딩 캐싱**: 메모리 기반 임베딩 캐싱으로 성능 향상 ⭐
- **에러 복구**: 벡터 검색 실패 시 키워드 검색으로 자동 폴백 ⭐

### 프로젝트 통계

- **Python 소스 파일**: 69개 (`src` 디렉토리)

### 레이어 구조

1. **API Layer** (`src/api/`)
   - FastAPI 기반 RESTful API
   - 라우터: search, ask, generate, admin, monitoring, health
   - 미들웨어: 로깅, Rate Limiting, CORS
   - 의존성 주입으로 서비스 재사용

2. **Service Layer** (`src/rag/`)
   - RAG 워크플로우 (LangGraph)
   - 콘텐츠 생성 워크플로우
   - 하이브리드 검색 (벡터 + 키워드 폴백)
   - FilterManager: 통합 필터링 관리 ⭐
   - 임베딩 생성기 (캐싱 지원) ⭐

3. **Data Layer**
   - 벡터 DB (ChromaDB)
   - 임베딩 생성 (OpenAI, 캐싱)
   - 세션 관리 (Redis 선택사항)

4. **Model Layer** (`src/models/`)
   - Pydantic 기반 데이터 모델
   - 10가지 문서 타입별 모델

## 핵심 모듈 상세 분석

### 1. API 레이어 (`src/api/`)

#### 1.1 메인 애플리케이션 (`main.py`)
- FastAPI 앱 생성 및 설정
- 미들웨어 등록 (CORS, Rate Limiting, Logging)
- 라우터 등록
- 예외 핸들러 등록
- 생명주기 관리 (startup/shutdown)

#### 1.2 검색 라우터 (`routers/search.py`)
- POST/GET `/api/v1/search`: 문서 검색
- 하이브리드 검색 (벡터 + 키워드)
- 메타데이터 필터링
- 쿼리 캐싱 (LRU, TTL)
- 문서 타입 필터링

#### 1.3 질의응답 라우터 (`routers/ask.py`)
- POST `/api/v1/ask`: 질의응답
- POST `/api/v1/ask/stream`: 스트리밍 질의응답
- RAG 기반 답변 생성
- 세션 관리 (멀티 턴 대화)
- 출처 정보 제공

### 2. RAG 레이어 (`src/rag/`)

#### 2.1 FilterManager (`filter_manager.py`) ⭐ 신규
**목적**: 필터링 로직 중앙화 및 코드 중복 제거

**주요 기능**:
- 문서 타입 필터링
- 메타데이터 필터링 (category, sub_category, court 등)
- 필터 완화 (결과가 없을 때)
- 쿼리 기반 필터 자동 추출

**효과**:
- 코드 중복 제거
- 필터링 로직 일관성 향상
- 유지보수성 개선

#### 2.2 임베딩 생성기 (`embedding.py`) ⭐ 개선
**개선 사항**: 메모리 기반 임베딩 캐싱 추가

**주요 기능**:
- 단일 텍스트 임베딩 (캐싱 지원)
- 배치 텍스트 임베딩 (동적 배치 크기)
- 캐시 히트율 향상으로 API 호출 감소
- 배치 처리 최적화

**성능 개선**:
- 동일 텍스트 재사용 시 API 호출 제로
- 배치 크기 자동 조정 (최대 100)

#### 2.3 하이브리드 검색기 (`retriever.py`) ⭐ 개선
**개선 사항**: 에러 복구 로직 추가

**주요 기능**:
- 벡터 검색 + 키워드 검색 결합
- 벡터 검색 실패 시 키워드 검색으로 폴백
- 메타데이터 필터링
- RAG 워크플로우 통합

**에러 복구**:
- `VectorStoreError` 또는 `EmbeddingError` 발생 시
- 자동으로 키워드 검색으로 폴백
- 폴백 사용 여부를 결과에 표시

#### 2.4 RAG 워크플로우 (`workflow.py`)
**LangGraph 기반 워크플로우**:
1. `analyze_query`: 쿼리 분석 및 임베딩 생성
2. `vector_search`: 벡터 검색 수행
3. `filter_metadata`: FilterManager를 통한 필터링 ⭐
4. `rerank_results`: 결과 재랭킹
5. `build_context`: 컨텍스트 구성

**주요 기능**:
- 쿼리에서 필터 자동 추출 (법률명, 조문번호, 사건번호 등)
- 문서 타입 자동 추론
- 같은 문서의 청크 병합
- 거리 기반 재랭킹

#### 2.5 콘텐츠 생성 워크플로우 (`content_workflow.py`)
**LangGraph 기반 콘텐츠 생성 워크플로우**:
1. `search_documents`: 관련 문서 검색
2. `summarize_documents`: 문서 요약
3. `generate_prompt`: 프롬프트 생성
4. `generate_draft`: 초안 생성
5. `evaluate`: 평가 (구조, 내용, 법적 정확성)
6. `comprehensive_evaluate`: 종합 평가
7. `rewrite_topic`: 주제 재작성 (필요 시)
8. `finalize_content`: 최종 콘텐츠 완성

**주요 기능**:
- 3단계 평가 시스템 (구조 30점, 내용 40점, 법적 정확성 30점)
- 자동 재작성 루프 (최대 3회)
- 프롬프트 조정 및 재검색
- 섹션별 구조화된 콘텐츠 생성

### 3. 모델 레이어 (`src/models/`)

#### 3.1 BaseDocument (`base.py`)
- 공통 필드: id, category, sub_category, type, title, content, metadata
- Pydantic BaseModel 상속
- 타입 검증 자동화
- JSON 직렬화 지원

#### 3.2 문서 타입별 모델
- **StatuteModel**: 법령 (law_name, article_number 등)
- **CaseModel**: 판례 (case_number, court 등)
- **ProcedureModel**: 절차 매뉴얼
- **TemplateModel**: 템플릿
- **ManualModel**: 실무 매뉴얼
- **StatisticsModel**: 통계 데이터
- 기타 4가지 타입

### 4. 유틸리티 레이어 (`src/utils/`)

## 데이터 흐름 분석

### 1. 데이터 인덱싱 흐름

```
JSON 파일
    ↓
[1] 파일 읽기 → JSON 파싱
    ↓
[2] 데이터 검증 → Pydantic 모델 변환
    ↓
[3] 데이터 정제 → 정제된 문서
    ↓
[4] 텍스트 청킹 → 청크 배열
    ↓
[5] 임베딩 생성 (캐싱) → 벡터 배열 ⭐
    ↓
[6] 벡터 DB 저장 → 인덱싱 완료
```

### 2. 검색 흐름

```
사용자 쿼리
    ↓
[1] 쿼리 분석 → 임베딩 생성 (캐싱) ⭐
    ↓
[2] 벡터 검색 → 유사 문서 검색
    ↓ [실패 시 폴백] ⭐
    └─→ 키워드 검색 (폴백)
    ↓
[3] FilterManager 필터링 ⭐
    ├─ 문서 타입 필터
    └─ 메타데이터 필터
    ↓
[4] 재랭킹 → 거리 기반 정렬
    ↓
[5] 컨텍스트 구성 → 청크 병합
    ↓
검색 결과 반환
```

### 3. 질의응답 흐름

```
사용자 질문
    ↓
[1] 세션 관리 → 대화 히스토리 로드
    ↓
[2] 검색 수행 → 관련 문서 검색 (폴백 지원) ⭐
    ↓
[3] 컨텍스트 구성 → 문서 + 히스토리
    ↓
[4] LLM 호출 → 답변 생성
    ↓
[5] 후처리 → 출처 추출, 스타일 검증
    ↓
[6] 세션 업데이트 → 대화 저장
    ↓
응답 반환 (사용자 친화적 에러 처리) ⭐
```

3. **확장성**: 새로운 문서 타입 및 기능 추가 용이
