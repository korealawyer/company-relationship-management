# 🏛️ IBS CRM — 개인고객 포탈 & 나홀로 소송 분리 설계 마스터 프롬프트
> **Phase 1 목표**: 로펌 CRM 내 완벽한 의뢰인 경험(UX) 통합 구축  
> **Phase 2 비전**: B2C 전용 "나홀로 소송 (Pro-se) 통합 플랫폼"으로 분리 독립 (SaaS화)  
> **분석 기준**: 2026 트렌드 기반 UX 원칙 (Zero-anxiety) + GAP 분석 2026-03-25 통합  
> **핵심 설계 원칙**: 의뢰인 포털(`/client-portal`)은 철저히 모듈화되어, 추후 단독 앱으로 스핀오프 가능하도록 `personalStore`를 직접 참조하지 않고 API/Props 기반으로 데이터 통신.

## 🌟 제품 비전 & 의뢰인 감성 UX (Zero-Anxiety Design)

| 📌 핵심 원칙 | 설명 | 구현 목표 |
|---|---|---|
| **데이터 무기화** | 경쟁사가 따라올 수 없는 결과 예측 모델 | 누적 판례 기반 "가사사건 유사 승소율 71% / 평균 4.2개월" 표시 |
| **Zero Anxiety** | 의뢰인의 극도 불안감을 시각적 안정감으로 치환 | "소장접수 ─●─ 1차변론 ─○─ 판결" 직관적인 Journey Map |
| **프리미엄 감성** | 변호사의 신뢰도와 IT 혁신성을 동시 전달 | 인디고-퍼플(`#4f46e5`) 기반 세련된 그라데이션 UI |
| **Invisible Complexity** | 30여 개 기능 중 첫 화면엔 지금 필요한 딱 1개만 | "이번 주 1차 변론 기일입니다." 액션 유도 배지 |
| **Celebration** | 승소/합의/종결 시 긍정적 각인 (바이럴 모멘텀) | 완료 시 화면 전체 축하 애니메이션 (🎉) |

자세한 독립 앱 설계 명세는 `client_portal_spec.md` 아티팩트를 참조합니다.

## 📊 A. 핵심 (P0) - 즉시 구현 필요 항목 (이번 주 내)

### A-1. AddCaseModal 입력 항목 구체화
- **현재**: `title`, `description` 등 단순 텍스트 입력만 가능.
- **요구**: `AddCaseModal`의 기일 등록 로직을 Select 드롭다운(변론기일/준비기일 등) 및 불변기일 토글(🔴)로 교체. 법정 호수 추가.

### A-2. DetailPanel 타임라인 컴포넌트 추가
- **현재**: 단순 텍스트 나열.
- **요구**: 과거 → 현재 → 예정 일정의 시각적 타임라인 UI 컴포넌트 추가.

### A-3. 이해충돌(Conflict Check) 기능 누락 
- **현재**: `checkConflict()` 로직 없음.
- **요구**: 신규 사건 등록 시 `personalStore.getAll()`을 순회하여 원고/피고 이름 중복 검사 로직 추가.

### A-4~6. Supabase 연동, 변호사 뱃지, 자동화 엔진
- 기업 포탈과 완벽히 동일한 수준으로 Supabase 연동(`sbSyncLatest`), 사이드바 뱃지 팝업, `runPersonalDeadlineAlerts` 추가 구현 필요.

## 📈 B. 부가 (P1) - 기능 확장 (2주 내)

### B-1. 통합 캘린더 데이터 머지
- 송무 캘린더와 개인 캘린더 데이터를 하나의 뷰에서 병합 출력.

### B-2. 문서함 권한 관리 (`DocVisibility`)
- 변호사 전용(`lawyer_only`) vs 의뢰인 열람 가능 문서 구분.

### B-4. 의뢰인 포털 (`/my-cases`) Magic Link + 온보딩 UX
- 의뢰인 전용 Magic Link 발송, Journey Map 진행바, 다가오는 출석일, 변호사 다이렉트 소통 채널.

### B-7. 분할 납부 관리 및 결제 연동
- 의뢰인 포털에 토스/카카오페이 원클릭 결제 버튼 추가 및 납부 현황 동기화.
