# 🔥 해지 방어 & 복구 워크플로우 (Churn Defense)

로펌 SaaS 고객의 이탈 위험을 조기에 감지하고, 단계별 개입으로 해지를 방지합니다.

---

## Step 1: 헬스스코어 자동 계산 (매일 06:00)
// turbo
```
(Supabase Edge Function: pg_cron 매일 06:00)
```

헬스스코어 계산 공식 (0~100점):

| 지표 | 가중치 | 측정 방법 |
|---|---|---|
| 최근 7일 로그인 빈도 | 30% | 로그인 수 / 기대 로그인 수 |
| 핵심 기능 사용 여부 | 25% | 이달 자동화 실행 건수 |
| 데이터 증가량 | 20% | 이달 사건/의뢰인 신규 등록 수 |
| 팀원 활성 비율 | 15% | 활성 팀원 / 전체 등록 팀원 |
| 지원 티켓 미해결 수 | 10% | 미해결 티켓 수 (반비례) |

등급 분류:
- 🟢 **Healthy** (80~100): 이탈 위험 낮음
- 🟡 **At Risk** (50~79): 모니터링 강화
- 🔴 **Critical** (0~49): 즉시 개입 필요

---

## Step 2: Critical 감지 → 즉시 개입 (자동 알림 후 수동 처리)

TRIGGER: 헬스스코어 < 50점 OR 전주 대비 -20점 이상 하락

자동 실행:
1. CS 담당자에게 Slack/이메일 즉시 알림
   ```
   🚨 [로펌명] 이탈 위험!
   헬스스코어: [N]점 → 전주 대비 [-M]점
   최근 7일 로그인: [N]회
   즉시 체크인 전화 필요
   ```
2. CS 담당자 To-Do에 "긴급 체크인 전화" 등록
3. `law_firms.health_status = 'at_risk'` 업데이트

---

## Step 3: 체크인 전화 (24시간 이내)

CS 담당자 전화 스크립트:
```
"안녕하세요, [로펌명] [대표변호사명] 변호사님.
저는 [플랫폼명]의 [CS 담당자명]입니다.
최근 플랫폼 사용에 혹시 불편하신 점이 있으신지 확인 드리려고 연락드렸습니다.

[침묵 + 경청]

(불편 사항 파악 후):
말씀해 주셔서 감사합니다.
바로 [조치 방법]으로 해결해 드리겠습니다.
다음 [날짜]까지 처리 결과 알려드릴게요."
```

전화 결과 기록:
- CRM law_firms.notes에 통화 내용 기입
- 감지된 이탈 이유 카테고리 선택:
  - [ ] 사용 방법 모름 (교육 필요)
  - [ ] 특정 기능 누락 (PM에게 전달)
  - [ ] 경쟁사로 전환 검토 중
  - [ ] 예산 이슈 (요금제 조정 검토)
  - [ ] 담당자 퇴직/변경 (재온보딩 필요)
  - [ ] 사용량 감소 (로펌 규모 축소)

---

## Step 4: 맞춤형 구제 조치 (이탈 이유별)

### Case A: 사용 방법 모름
- 1:1 화면공유 재교육 세션 예약 (30분)
- 핵심 기능 사용 가이드 재발송
- "빠른 시작 비디오" 맞춤 링크 발송

### Case B: 기능 누락
- PM에게 즉시 피처 리퀘스트 전달
- 유사 기능 대안 안내 (현재 가능한 방법)
- 기능 출시 예정 일정 공유 (가능 시)

### Case C: 경쟁사 전환 검토
- 경쟁사 대비 우리 강점 정리 자료 발송
- 업그레이드 제안 또는 요금 협상 카드
- CEO가 직접 전화 (대형 고객의 경우)

### Case D: 예산 이슈
- 요금제 다운그레이드 옵션 안내
- 연간 결제 전환 시 월 납부 대비 절감액 계산 공유
- ❌ 50% 이상 할인은 대표 승인 필요

### Case E: 담당자 변경
- 신규 담당자에게 온보딩 세션 재진행
- `_agents/workflows/saas_demo.md` Step 5 체크리스트 재실행

---

## Step 5: 해지 요청 접수 시 최후 방어 (Win-Back)

해지 접수 즉시:
1. "해지 신청 완료" 자동 이메일 발송 (즉각 응답 선호)
2. 해지 이유 설문 자동 발송 (3문항 이내)
3. CS 책임자가 1일 이내 "마지막 협의" 전화

Win-Back 제안 카드 (순서대로 사용):
```
1차: "해지 전 한 달만 더 사용해 보세요. 1개월 무료 연장."
2차: "기능 [X]가 다음 달에 출시됩니다. 조금만 기다려 주세요."
3차: "Pro → Basic으로 다운그레이드 후 6개월만 더 사용해 보시겠어요?"
```

---

## Step 6: 해지 완료 처리 (방어 실패 시)

자동 처리:
- `law_firms.status = 'churned'` 업데이트
- 데이터 보관 모드 전환 (90일 유지 후 삭제)
- 데이터 백업 다운로드 URL 이메일 발송
- `MRR` 감소 반영 → `_agents/finance.md` 트래커 업데이트

해지 이유 분석 → PM에게 전달:
- 해지 이유 카테고리 집계
- 동일 이유 3건 이상 누적 → 제품 개선 아이템으로 자동 등록

- [ ] Critical 등급 24시간 이내 체크인 전화 완료
- [ ] 맞춤형 구제 조치 실행 완료
- [ ] 해지 방어 성공 or 해지 완료 처리
- [ ] 해지 이유 PM + CEO에게 전달
