# /저장 워크플로우

> ⚠️ **절대 규칙**: @참조 파일의 폴더 위치는 저장 위치와 **무관**. 타입으로만 결정.
> 예: `@ceo`를 참조했어도 내용이 SEO이면 → `_docs/seo/`에 저장.

---

## Step 1: 타입 판단

대화 내용을 분석하여 아래 타입 중 하나로 분류한다.

| 타입 키워드 | 판단 기준 | 저장 위치 |
|---|---|---|
| SEO | 키워드, 검색 순위, 네이버/구글, 백링크, 메타태그 | `_docs/seo/` |
| SALES | 전환율, 퍼널, 영업 스크립트, 고객 획득 | `_docs/sales/` |
| FEATURE | 기능 기획, 페이지 구조, PRD, 유저 스토리 | `_docs/features/` |
| OPS | 자동화, 배포, CRM, 시스템 운영, API | `_docs/ops/` |
| GROWTH | 그로스 해킹, 채널별 성장, 블로그/SNS 전략, 바이럴 | `_docs/growth/` |
| STRATEGY | 의사결정, 방향성, 사업 전략, 투자 판단 | `_strategy/` |
| LOG | 대화 요약, 아이디어 메모, 회의록 | `_logs/` |
| AGENT | 프롬프트, 에이전트 설정, AI 규칙 | `_agents/` |

---

## Step 2: 파일명 생성 원칙

사용자가 파일명을 지정하지 않은 경우:
1. 핵심 주제 영어 단어 2~4개 조합
2. 언더스코어 구분
3. 날짜 불필요 (폴더 내 정렬은 이름으로)

```
예시:
- SEO 프랜차이즈 전략 → seo_franchise_strategy.md
- CRM 대시보드 기획 → crm_dashboard_prd.md
- 고객 전환 퍼널 → sales_conversion_funnel.md
- 네이버 블로그 그로스 → growth_naver_blog.md
```

---

## Step 3: 저장 전 타입 확인 (1줄 확인)

저장 전 반드시 한 줄로 확인:

```
"[타입]으로 분류 → [경로/파일명]에 저장할게요. 맞나요? (맞으면 Enter, 다르면 수정 지시)"
```

예:
```
FEATURE로 분류 → _docs/features/crm_dashboard_prd.md에 저장할게요. 맞나요?
```

---

## Step 4: 사용자 경로 지정이 있으면 그것을 따른다

```
/저장 _docs/seo/                     → 해당 폴더, 파일명 자동
/저장 _docs/features/dashboard.md    → 경로 + 파일명 모두 지정
/저장 _strategy/                     → strategy 폴더에 저장
```

---

## Step 5: 저장 후 확인 메시지

저장 완료 시 아래 형식으로 보고:

```
저장 완료 ✅
📁 경로: _docs/seo/seo_franchise_strategy.md
📌 타입: SEO 전략
```

---

## Step 6: 모호한 경우 처리

하나의 대화에 복수 타입이 섞여 있으면:
- 주요 타입 기준으로 저장
- 사용자에게 "FEATURE + SEO 혼합 - FEATURE로 분류했습니다" 고지
- 유형 판단 불가 시 → `_logs/misc_[날짜].md` 저장 후 확인 요청

---

## 명령어 예시 모음

```bash
/저장                                      # 자동 판단 + 1줄 확인
/저장 _docs/seo/                           # 폴더 지정
/저장 _docs/features/crm_dashboard.md     # 경로+파일명
/저장 파일명: my_strategy                  # 파일명만 지정
/저장 _logs/                              # 대화 로그 저장
```
