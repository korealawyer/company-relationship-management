# /newsletter_dispatch 워크플로우

> **트리거**: 스케줄 자동 실행 (Cron) or 수동 `/newsletter_dispatch` 명령
> **목표**: 변호사가 읽고 승인 버튼 1번만 누르면 완료되는 뉴스레터 발송
> **참고 문서**: `_strategy/FRANCHISE_NEWSLETTER_SYSTEM.md`, `_agents/newsletter.md`

---

## STEP 1. 발송 대상 & 콘텐츠 확인

오늘 날짜 기준으로 발송해야 할 콘텐츠 자동 확인:

```
발송 캘린더 체크:
  - 매주 월요일 → B4 개폐업 시그널 트래커
  - 격주 화요일(홀수주) → A2 공정위 처분 동향
  - 격주 목요일(짝수주) → B2 트렌드 리포트 (ENTERPRISE)
  - 매월 1일 → A1 법령 뉴스레터 + B5 노무 이슈
  - 매월 5일 → B1 경영지표 리포트
  - 매월 10일 → A4 개인정보 스코어카드
  - 매월 15일 → B6 경쟁사 인텔리전스
  - 공정위 처분 감지 즉시 → A5 실시간 처분 알림
  - 분기 첫째 주 월요일 → A6 분기 법무 헬스체크
```

발송 대상 의뢰인 필터링:
- `corporate_clients` 테이블에서 `status = 'active'` 조회
- `contract_tier` 기준으로 해당 콘텐츠 수신 여부 확인 (BASIC/PRO/ENTERPRISE)
- 발송 중단 플래그(`newsletter_paused = true`) 제외

---

## STEP 2. 데이터 수집

각 콘텐츠 유형별 데이터 소스에서 최신 데이터 자동 수집:

```
A1 (법령): law.go.kr 입법예고 RSS → 지난 30일 개정·예고 수집
A2 (처분): 공정위 가맹사업 처분 DB → 지난 14일 처분 수집
A3 (정보공개서): franchise.ftc.go.kr → 의뢰인 정보공개서 최신판 파싱
A4 (개인정보): 의뢰인 홈페이지 크롤링 → 처리방침 텍스트 추출
A5 (실시간): 공정위 RSS 실시간 폴링 → 가맹 관련 처분 감지
A6 (분기): 전분기 A1~A5 누적 데이터 집계
B1 (경영지표): 통계청 API, 소상공인진흥공단 데이터
B2 (트렌드): 네이버 DataLab API, 뉴스 크롤링
B3 (준수체크): 의뢰인 가맹계약서 DB (업로드된 파일 활용)
B4 (개폐업): 공정위 정보공개서 가맹점 수 주간 델타
B5 (노무): 고용노동부 행정해석 DB, 최저임금위원회 공시
B6 (경쟁사): 뉴스 API, 공정위 DB, 구인공고 크롤링
```

데이터 수집 실패 시:
- 재시도 3회 후 실패 → 해당 섹션 skip + 변호사에게 수동 보완 요청 표시
- 전체 데이터 수집 실패 → 발송 지연 + 슬랙 알림

## STEP 3. AI 초안 생성

의뢰인별 맞춤 초안 자동 생성 (LLM 호출):

```
입력:
  - 의뢰인 프로파일 (brand_name, industry, franchise_count, region, risk_flags)
  - 수집된 원본 데이터
  - 콘텐츠 유형별 프롬프트 템플릿 (_agents/newsletter.md PART 2 참조)
  - 계약 티어 (BASIC/PRO/ENTERPRISE)

출력:
  - HTML 이메일 버전 (인라인 CSS)
  - 텍스트 버전 (이메일 대체)
  - 카카오 요약 버전 (90자 이내, A5 전용)
  - 품질 체크 결과 (자동 검증 통과 여부)
```

품질 자동 검증 (PASS/FAIL):
  [ ] 브랜드명 최소 3회 이상 등장
  [ ] 법령 조항 번호 형식 유효 (A 계열만)
  [ ] 과태료 금액 법령 원문 교차 확인 (A 계열만)
  [ ] 발송일 기준 30일 이내 데이터 사용
  [ ] 분량 기준 충족 (이메일 8분 이내 완독)

FAIL 시 → 재생성 1회 → 재검증 → 2회 연속 FAIL → 담당자 수동 검토 요청

## STEP 4. 변호사 컨펌 큐 등록

법률 콘텐츠 (A1, A2, A3, A4, A5, A6, B3, B5)는 컨펌 큐 등록:

```
컨펌 큐 등록:
  - /lawyer 페이지 "뉴스레터" 탭에 초안 표시
  - 슬랙 #newsletter-confirm 자동 메시지 발송:
    "[콘텐츠 코드] [의뢰인수]개사 뉴스레터 컨펌 필요
     예상 소요 시간: [N]분 | 예정 발송: [날짜 시간]"
  - 담당 변호사 카카오 알림 발송

컨펌 타임아웃:
  - A5 (긴급 알림): 2시간 내 미컨펌 → 에스컬레이션
  - A1, A2, B5: 발송 예정 6시간 전까지 컨펌 필요
  - A3, A4, A6, B3: 발송 예정 24시간 전까지 컨펌 필요
  - 타임아웃 시 → 슬랙 + 문자 에스컬레이션

컨펌 인터페이스 (변호사 화면):
  ┌─────────────────────────────────────────┐
  │ 📧 [콘텐츠명] — [의뢰인명]             │
  │ 데이터 기준일: YYYY-MM-DD              │
  │ 예정 발송: YYYY-MM-DD HH:00            │
  │                                         │
  │ [모바일 미리보기] [PC 미리보기]         │
  │                                         │
  │ ✅ 승인 (즉시 발송 큐 진입)            │
  │ ❌ 반려 사유: [법적 부정확 / 데이터    │
  │   오류 / 톤 부적절 / 기타]             │
  └─────────────────────────────────────────┘

반려 시:
  → 사유 기록 후 재생성 트리거
  → 개선된 프롬프트로 재생성 (수정 요청 아님, 재시도)
  → 재생성 결과 컨펌 큐 재등록
```

경영 콘텐츠 (B1, B2, B4, B6):
  - 컨펌 큐 등록 없이 STEP 5로 바로 진행

---

## STEP 5. 자동 발송

변호사 승인(또는 컨펌 불필요 콘텐츠) 즉시 자동 발송:

```
이메일 발송 (Resend / SendGrid):
  - HTML 이메일 + 텍스트 대체
  - 발신자: "IBS 법무팀 <newsletter@ibslawfirm.kr>"
  - 수신자: 의뢰인 CRM 등록 이메일 (담당자 + CC 목록)
  - 제목: [콘텐츠별 자동 생성 제목]
  - UTM 파라미터 자동 추가 (열람 추적용)

카카오 알림톡 (A5 실시간 알림 전용):
  - 수신: CRM 등록 담당자 휴대폰
  - 내용: 90자 요약 + [검토 요청 / 아니오] 버튼
  - 버튼 클릭 → /client-portal 해당 리포트로 딥링크

포털 업로드:
  - /client-portal 내 "리포트 보관함" 자동 업로드
  - 파일명: [YYYY-MM-DD]_[콘텐츠코드]_[브랜드명]
  - 읽음 여부 추적 활성화
```

## STEP 6. CRM 기록 & 성과 추적

발송 완료 후 자동 기록:

```
CRM 업데이트:
  - newsletter_history 테이블: 발송일, 콘텐츠 코드, 의뢰인, 변호사 컨펌 여부
  - 이메일 오픈 이벤트: 추적 픽셀 기반 자동 기록
  - 클릭 이벤트: UTM 기반 자동 기록
  - 포털 열람: 접속 로그 자동 기록

발송 결과 알림 (슬랙 #newsletter-sent):
  "[날짜] [콘텐츠명] 발송 완료
   총 N개사 | 오픈율 XX% (24시간 후 업데이트)
   변호사 컨펌 소요: N분"

월간 성과 리포트 (매월 말일 자동 생성):
  - 콘텐츠별 평균 오픈율 / 클릭률
  - 의뢰인별 열람 패턴 (해지 리스크 조기 감지)
  - 컨펌 거절율 (초안 품질 지표)
  - 다음 달 개선 포인트 추천 (갱신율 상관관계 분석)
```
