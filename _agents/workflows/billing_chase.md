# 💸 미납 수금 추적 워크플로우

수임료 미납 발생 시 D+1부터 D+60까지 자동으로 단계별 수금 조치를 실행합니다.

---

## Step 0: 미납 감지 (매일 09:00 자동)
// turbo
```
(Supabase Edge Function: pg_cron 매일 09:00)
SELECT * FROM billing 
WHERE due_date < CURRENT_DATE 
  AND status = 'pending'
  AND law_firm_id = [FIRM_ID]
```

조건: `billing.due_date < 오늘` AND `billing.status = 'pending'`
ACTION:
- `billing.status = 'overdue'` 자동 업데이트
- 담당 변호사 대시보드 빨간 배지 표시
- D+1 액션 트리거

---

## Step 1: D+1 — 1차 자동 알림 (카카오/이메일)

자동 발송:
```
[알림] [법무법인명] 수임료 납부 안내

[의뢰인명] 님, 안녕하세요.
[청구번호 INV-XXXXXX]의 납부 기한(YYYY-MM-DD)이 지났습니다.

청구금액: [금액]원
납부계좌: [은행] [계좌번호]

빠른 납부 감사드립니다.
문의: [담당 변호사명] (010-XXXX-XXXX)
```

기록: automation_logs 테이블에 발송 기록 저장

---

## Step 2: D+7 — 2차 알림 + 담당 변호사 알림

자동 실행:
1. 의뢰인에게 카카오 알림 재발송 (톤 강화)
2. 담당 변호사에게 알림: "미납 7일 경과 — [의뢰인명] [금액]원"
3. 담당 변호사 To-Do에 "전화 독촉" 자동 등록

담당 변호사 액션 (수동):
- 의뢰인에게 직접 전화
- 분납 희망 시 분납 계획서 작성 후 CRM 기록
- `billing.notes`에 통화 결과 기입

---

## Step 3: D+14 — 법적 조치 검토 알림

자동 실행:
1. 대표 변호사에게 에스컬레이션 알림
2. 내용증명 발송 검토 알림 (케이스별 판단)
3. billing.status = 'at_risk' 업데이트

담당 변호사 + 대표 변호사 협의:
- 분납 합의 → billing에 분납 스케줄 등록
- 계속 미납 → 내용증명 발송 결정
- 사건 취소 가능성 → 사건 팀 내부 회의

내용증명 발송 시:
```
[내용] [의뢰인명] 귀중
귀하의 [사건명] 관련 수임료 [금액]원이 [납부기한]까지 미납 상태입니다.
이 서신 수령 후 7일 이내 납부하지 않을 경우, 법적 조치를 취할 예정입니다.
```

---

## Step 4: D+30 — 강경 조치 & bad_debt 처리

자동 실행:
1. `billing.status = 'bad_debt'` 플래그
2. 담당 변호사에게 "수임 계약 해지 검토" 알림
3. 의뢰인 신용 등급 → 'blocked' 설정 (재수임 주의 표시)

수동 처리:
- 지급명령 신청 검토 (소액 청구 시 간이 절차)
- 수임계약 해지 통보 (잔여 착수금 처리 협의)
- 미납 금액 대손처리 (회계팀 통보)

---

## Step 5: D+60 — 최종 법적 조치

대표 변호사 최종 결정:
- 소송 제기 여부 결정 (착수금 미납 + 법무 서비스 제공한 경우)
- 소비자 분쟁조정 위원회 신청
- 신용정보 제공 동의 여부 확인 후 연체 등록

CRM 처리:
- cases.status = 'dispute' 업데이트 (내부 분쟁 사건으로 등록)
- 해당 의뢰인 재수임 시 대표 변호사 승인 필수 플래그

---

## SaaS 로펌 구독료 미납 (Track B)

구독료 미납 발생 시 별도 프로세스:
- D+3: 자동 이메일 + 결제 링크 재발송
- D+7: 계정 기능 일부 제한 (읽기 전용 모드 전환)
- D+14: 계정 일시 정지 (데이터 삭제 아님)
- D+30: 계정 해지 경고 이메일
- D+45: 최종 해지 처리, 데이터 90일 보관 → 백업 URL 발송
