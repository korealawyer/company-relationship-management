## 🧬 한국 3대 디자인 시스템 마스터리

### 1. 카카오(Kakao) 디자인 DNA — "친근한 신뢰"
설계 철학:
  - 둥근 모서리(border-radius: 12~16px)로 부드러운 인상
  - 카카오 옐로(#FEE500)는 쓰지 않지만, "따뜻한 중립" 톤 계승
  - 시스템 폰트 Apple SD Gothic Neo → 한국인이 가장 편안하게_느끼는 서체
  - 대화형 인터페이스 패턴 (카카오톡 UX가 한국인 기본 멘탈 모델)
  - 이모지 + 한글 레이블 조합 → 직관적 네비게이션

적용 포인트:
  ✅ 채팅/메시지형 UI → 법률 상담 로그, 코멘트 스레드
  ✅ 카드형 레이아웃 → 사건 카드, 의뢰인 프로필
  ✅ 바텀시트 패턴 → 모바일 퀵 액션 (승인, 서명)
  ✅ 토스트 알림 → 작업 완료/에러 피드백

### 2. 네이버(Naver) 디자인 DNA — "정보 밀도의 정석"
설계 철학:
  - 정보 밀도가 높아도 읽기 쉬운 구조 (그리드 + 카드 + 탭)
  - 네이버 그린(#03C75A)은 쓰지 않지만, "깔끔한 정돈" 감각 계승
  - 탭/세그먼트 컨트롤로 복잡한 정보를 분류
  - 검색 중심 UX → 한국인의 검색 습관에 최적
  - 리스트+필터 조합 → 대량 데이터 탐색 패턴

적용 포인트:
  ✅ 복합 대시보드 → KPI 카드 + 테이블 + 차트 밀도 있는 배치
  ✅ 탭 기반 정보 아키텍처 → 사건/의뢰인/문서 전환
  ✅ 강력한 검색+필터 → 사건 검색, 의뢰인 검색
  ✅ 댓글/리뷰 스타일 → 문서 코멘트 시스템

### 3. 토스(Toss) 디자인 DNA — "미니멀 임팩트"
설계 철학:
  - 한 화면에 하나의 액션만 강조 (Single CTA 원칙)
  - 숫자가 주인공 — 금액·통계를 가장 큰 폰트로 배치
  - 마이크로 인터랙션으로 "살아있는" 느낌
  - 다크모드 기본 지원 (야간 업무 변호사 배려)
  - 스켈레톤 로딩 → 체감 속도 극대화
  - 바텀 네비게이션 → 모바일 한 손 조작

적용 포인트:
  ✅ 금융형 숫자 UI → 수임료, 매출, 미수금 대시보드
  ✅ 원탭 승인 → 문서 승인, 계약 서명
  ✅ 단계별 온보딩 → 신규 로펌 셋업 위자드
  ✅ 진행률 UI → 사건 진행 상태 바, 파이프라인 스텝

## 🇰🇷 한국인 신뢰 디자인 코드 (심리 기반)

### 신뢰 시그널 5원칙:

1. **"공식적이다" 시그널** (Authority)
   - 네이비(#1B2559) + 딥블루(#0A1628) 기반 컬러
   - 법원/정부 공식 문서 느낌의 레이아웃 참조
   - 인증 뱃지, 공식 로고 배치 (대한변호사협회 등)
   - 폰트: Pretendard (한국 IT 업계 표준) or SUIT (무료 한글 최적)
   
2. **"안전하다" 시그널** (Security)
   - 🔒 아이콘 + "암호화 보호" 라벨 (개인정보 입력 필드)
   - SSL 인증서 뱃지 푸터 표시
   - 비밀번호 강도 미터 (한국 사용자 필수 기대)
   - 개인정보 처리 방침 링크 항상 노출

3. **"실적이 있다" 시그널** (Social Proof)
   - 숫자 카운터 애니메이션: "1,247개 로펌이 사용 중"
   - 고객사 로고 스트립 (대형 로펌 우선 배치)
   - 별점 + 리뷰 카드 (네이버 플레이스 리뷰 스타일)
   - "서울 강남구 소재 XX법무법인" 같은 구체적 프로필

4. **"한국 맞춤이다" 시그널** (Localization)
   - 한국식 날짜 포맷: 2026년 3월 18일 (화)
   - 금액: ₩1,000,000 (천단위 콤마 + 원화 기호)
   - 전화번호: 02-1234-5678 (한국식 하이픈)
   - 한글 존댓말 UX 카피 필수 ("저장되었습니다", "확인하세요")
   - 카카오 로그인 버튼 (한국 최고 전환율)

5. **"전문가 전용이다" 시그널** (Exclusivity)
   - 법률 전문 용어 적극 사용 (쉬운말 X → 전문성 훼손)
   - 사건번호, 법원명, 변호사 등록번호 필드 네이티브 지원
   - 법원 전자소송 시스템 연동 시각적 표현
   - "변호사 전용 플랫폼" 배지

## 📐 컴포넌트 디자인 패턴

### 1. 대시보드 KPI 카드 (토스 스타일)
구조:
  ┌─────────────────────┐
  │ 📊 이번 달 수임료     │ ← 캡션 (gray-500, 13px)
  │ ₩ 47,500,000        │ ← 금액 (display, 36px, navy-800, 카운트업 애니메이션)
  │ ▲ 12.5% vs 지난달    │ ← 변동률 (success, 13px, 화살표 아이콘)
  └─────────────────────┘
  
규칙:
  - 숫자가 주인공 → 가장 큰 폰트
  - 양수=초록, 음수=빨강, 제로=회색
  - 로딩 시 스켈레톤 애니메이션 (shimmer)
  - 터치/클릭 시 상세 페이지 전환 (→ 리플 이펙트)

### 2. 사건 카드 (카카오 스타일)
구조:
  ┌──────────────────────────────────┐
  │ 🟢 진행중         2026-0042      │ ← 상태 뱃지 + 사건번호
  │                                  │
  │ (주)한국프랜차이즈 가맹계약 분쟁    │ ← 사건명 (h3, 18px, bold)
  │ 의뢰인: 홍길동 대표  │  담당: 김변호사 │ ← 관계자 (body, gray-700)
  │                                  │
  │ ⏰ D-14 마감     ₩5,000,000 미수  │ ← 핵심 지표 (warning/danger 컬러)
  │──────────────────────────────────│
  │ 📎 3  💬 7  ✅ 2/5              │ ← 문서·코멘트·체크리스트 카운트
  └──────────────────────────────────┘

규칙:
  - border-radius: 12px
  - hover: shadow-md → shadow-lg + translateY(-2px)
  - 상태 뱃지: 도트 + 텍스트 (진행중=green, 대기=yellow, 종결=gray)
  - D-day 14일 이내: warning 컬러, 7일 이내: danger 컬러 + 펄스 애니메이션

### 3. 테이블 (네이버 스타일)
구조:
  - 헤더: gray-100 배경 + gray-700 텍스트 + 정렬 아이콘
  - 행: hover 시 navy-50 배경 하이라이트
  - 줄무늬: 없음 (한국 앱은 호버 하이라이트 선호)
  - 페이지네이션: "이전 | 1 2 3 ... 10 | 다음" (네이버 스타일)
  - 빈 상태: 일러스트 + "등록된 사건이 없습니다" + CTA 버튼

### 4. 모바일 바텀시트 (토스+카카오 하이브리드)
구조:
  - 드래그 핸들 바 (상단 중앙, 40px × 4px, gray-300)
  - 제목 + 닫기 버튼
  - 컨텐츠 (스크롤 가능)
  - 하단 CTA 버튼 (Safe Area 대응)
  - 백드롭: rgba(0,0,0,0.4)

적용:
  - 문서 빠른 승인
  - 사건 상태 변경
  - 수임료 빠른 입력
  - 카카오 알림 발송 확인

### 5. 폼 디자인 (한국 특화)
규칙:
  - 레이블: 인풋 위에 배치 (왼쪽 정렬, 13px, gray-700)
  - 필수 표시: 레이블 뒤 빨간 * (한국 사용자 기대)
  - 인풋: height 48px (모바일 터치 타겟)
  - 에러 메시지: 인풋 하단 빨간 텍스트 (13px)
  - 성공 피드백: 인풋 우측 체크 아이콘 (green)
  - 비밀번호: 보기/숨기기 토글 아이콘 (한국 필수)
  - 주민등록번호: 앞 6자리 + '-' + 뒤 7자리 (마스킹: ●●●●●●●)
  - 사업자등록번호: XXX-XX-XXXXX 자동 포맷
  - 전화번호: 010-XXXX-XXXX 자동 하이픈

### 6. 네비게이션 시스템
**데스크톱 — 사이드바:**
  - 너비: 240px (접힘: 64px)
  - 배경: navy-950
  - 아이콘 + 한글 레이블
  - 활성 메뉴: navy-800 배경 + gold-500 좌측 보더
  - 폴더형 메뉴 (사건관리 ▸ 민사 / 형사 / 기업법무)

**모바일 — 바텀 네비게이션:**
  - 5개 탭 제한 (홈, 사건, 문서, 알림, 더보기)
  - 활성: navy-800 아이콘 + 텍스트, 비활성: gray-500
  - 알림 뱃지: danger 컬러 원형 카운터
  - Safe Area padding-bottom 대응

## 🖥️ 페이지별 UX 설계 원칙

### 랜딩 페이지 (전환 최적화)
구성:
  1. 히어로 — 한 줄 핵심 가치 + CTA ("30일 무료 시작")
  2. 소셜 프루프 — 고객사 로고 + "1,247개 로펌 사용중"
  3. 기능 쇼케이스 — 스크롤 트리거 애니메이션 3~5개
  4. 가격 — 3단 가격표 (기본/프로/엔터프라이즈)
  5. 고객 후기 — 실제 변호사 이름 + 소속 (네이버 리뷰 스타일)
  6. FAQ — 아코디언 (6~8개)
  7. 최종 CTA — 풀 와이드 + "지금 시작하세요"

전환 디자인 규칙:
  ✅ CTA 버튼: gold-500 배경 + white 텍스트 + 호버 시 scale(1.02)
  ✅ 스크롤 시 헤더 고정 + 배경 블러(backdrop-filter: blur(12px))
  ✅ 모바일: 플로팅 CTA 버튼 (하단 고정)
  ✅ 로딩: 0.8초 이내 FCP (퍼포먼스 = 신뢰)

### 대시보드 (정보 밀도 최적화)
레이아웃:
  - 상단: 4열 KPI 카드 (토스 금액 스타일)
  - 중단 좌: 사건 현황 도넛 차트 + 최근 사건 리스트
  - 중단 우: 오늘의 일정 타임라인 + 마감 임박 사건
  - 하단: 최근 활동 피드 (네이버 카페 타임라인 스타일)

### 문서 뷰어 (3패널 레이아웃)
구성:
  [문서 목록] | [PDF/문서 뷰어] | [코멘트 스레드]
  
  - 좌측 패널: 문서 트리 (폴더식, 네이버 메일 스타일)
  - 중앙 패널: PDF 뷰어 + 인라인 주석 하이라이트
  - 우측 패널: 코멘트 스레드 (카카오톡 대화 스타일)
  - 모바일: 탭 전환 (문서 / 코멘트 / 승인이력)

## 📱 반응형 브레이크포인트

  --mobile:  0~767px     // 1열, 바텀 네비, 풀와이드 카드
  --tablet:  768~1024px  // 2열, 사이드바 접힘
  --desktop: 1025~1440px // 3열, 사이드바 펼침
  --wide:    1441px+     // 4열, 넓은 테이블

모바일 우선 원칙:
  ✅ 변호사의 60%는 법정/이동 중 모바일 사용
  ✅ 터치 타겟: 최소 44×44px (iOS HIG 기준)
  ✅ 한 손 조작 최적화 (중요 버튼은 화면 하단)
  ✅ 스와이프 제스처: 사건 카드 좌→우 = 빠른 액션

## 🔔 알림/피드백 UX 패턴

### 토스트 (Toast)
  - 위치: 상단 중앙 (데스크톱), 하단 (모바일)
  - 지속: 3초 → 자동 소멸 (슬라이드 아웃)
  - 성공: 체크 아이콘 + 녹색 좌측 보더
  - 에러: X 아이콘 + 빨간 좌측 보더
  - 실행 취소: "되돌리기" 텍스트 버튼 포함

### 빈 상태 (Empty State)
  - 중앙 정렬 일러스트 (64×64px)
  - 제목: "아직 등록된 사건이 없습니다"
  - 설명: "새 사건을 등록하고 업무를 시작하세요"
  - CTA: "+ 사건 등록" 버튼 (primary)

### 로딩 상태
  - 스켈레톤 UI (shimmer 애니메이션, 토스 스타일)
  - 무한 스크롤: 하단 스피너
  - 전체 페이지: 로고 + 프로그레스 바

## Hard Constraints:
❌ 카카오/네이버/토스 브랜드 컬러 직접 사용 금지 (법적 이슈)
❌ 반말/외래어 남발 금지 → 존댓말 + 법률 전문용어 사용
❌ 플레이스홀더 이미지/Lorem ipsum 금지 → 실제 법률 데이터로 디자인
❌ 복잡한 제스처 (핀치, 3D 터치) → 변호사는 간단한 조작 선호
✅ 모든 페이지 2초 이내 로드 (속도 = 신뢰)
✅ 모든 버튼/링크에 hover + active + focus 상태 정의
✅ 한글 폰트 렌더링 최적화 (font-display: swap, subset 적용)
✅ 프린트 스타일시트 별도 정의 (변호사는 프린트를 많이 함)
✅ 첫 화면에서 "이것은 전문가 도구"임을 3초 안에 인지

## PART 3: UX 라이팅 가이드 (한국어)

### 톤 앤 매너
- **존댓말(해요체)** 기본 → "저장했습니다", "확인해 주세요"
- **간결체** → 불필요한 수식어 제거
- **능동형** → "삭제됩니다" 대신 "삭제합니다"
- **구체적** → "잠시 후" 대신 "약 3초 후"

### 상황별 카피 예시

**성공:**
  ✅ "사건이 등록되었습니다"
  ✅ "문서가 성공적으로 업로드되었습니다"
  ✅ "수임료 ₩5,000,000이 수납 처리되었습니다"

**에러:**
  ❌ "네트워크 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."
  ❌ "필수 항목을 입력해 주세요."
  ❌ "권한이 없습니다. 관리자에게 문의하세요."

**확인:**
  ⚠️ "정말 삭제하시겠습니까? 이 작업은 되돌릴 수 없습니다."
  ⚠️ "송금을 진행하시겠습니까? 수임료 ₩5,000,000"

**빈 상태:**
  📭 "등록된 사건이 없습니다. 첫 사건을 등록해 보세요."
  📭 "이번 달 청구 내역이 없습니다."

**로딩:**
  ⏳ "사건 목록을 불러오는 중입니다..."
  ⏳ "문서를 준비하고 있습니다..."

## PART 4: 디자인 핸드오프 규칙

### 개발팀 협업 원칙
1. **CSS 커스텀 프로퍼티** — 모든 디자인 토큰은 CSS 변수로 전달
2. **컴포넌트 명세** — Props, States, Variants 명시
3. **모션 명세** — duration + easing + property 구체적 표기
4. **반응형** — 모바일 → 태블릿 → 데스크톱 순서로 디자인 제공
5. **인터랙션** — hover, active, focus, disabled, loading 모든 상태 포함

## PART 5: 경쟁 UI 분석 & 차별화

### 한국 법률 SaaS 경쟁사 UI 약점
| 경쟁사 | UI 약점 | 우리의 차별화 |
|---|---|---|
| 라인웍스(LineWorks) | 범용 업무 도구, 법률 특화 부족 | 법률 전문 컴포넌트 (사건카드, 승인바) |
| 케이스노트(CaseNote) | 기능 중심, 디자인 후순위 | 기능+디자인 동시 프리미엄 |
| 법률N(LawN) | 정보 과밀, 모바일 미지원 | 모바일 퍼스트 + 정보 밀도 최적화 |

### 핵심 차별화 전략
> "법률 소프트웨어인데 토스처럼 예쁘다"
> — 이것이 입소문의 트리거가 된다.

1. **첫 인상 3초 법칙** — 로그인 직후 KPI 애니메이션으로 "와우" 연출
2. **마이크로 인터랙션** — 모든 클릭에 햅틱 피드백 같은 시각 반응
3. **다크모드 기본** — 야간 작업 변호사 = 프리미엄 사용자
4. **프린트 완벽 지원** — 다른 법률 SaaS가 무시하는 영역
